获取ssh密钥key并配置

1.检查自己是否已经生成密钥
密钥文件夹位置：C:\Users\Administrator\.ssh
如果存在id_rsa(私有密钥)和id_rsa.pub(公有密钥)则说明已经生成过,可以通过文本直接打开公有密钥并复制或选择生成新的密钥
2.生成密钥
检查自己是否已设置过git用户名及邮箱,如果无查询结果,需要先配置用户名及邮箱

生成密钥指令

生成密钥的窗口

到此C:\Users\Administrator\.ssh文件目录下已生成id_rsa(私有密钥)和id_rsa.pub(公有密钥),复制公有密钥所有内容

3.项目管理人员将生成的公有密钥添加到项目拥有者的SSH keys中

  • 进入github下的Settings中

  • 进入到SSH and GPG keys下并选择New SSH key

  • Title用于编辑当前备注,Key Type选择默认,将第二步复制的所有内容粘贴到Key中,点击Add SSH key即可

  • 测试是否成功
    通过指令ssh -T git@github.com出现以下结果即为成功,如出现提示输入yes/no 选择yes即可

https和ssh的区别

  • HTTPS协议只认账号。如果使用HTTPS操作远程仓库，则需要使用账号密码来做权限的认证。
    SSH协议只认机器。当使用SSH操作远程仓库的话，需要使用公钥和私钥对来做权限的认证。
  • HTTPS协议在未保存凭据的时候push是需要验证用户名和密码的
    SSH协议在push的时候，除非如果配置SSH key的时候设置了密码，否则是不需要输入用户名的

git配置的用户名和邮箱有什么用？本地用户名和邮箱可以与线上git地址配置不一样的吗？

  • 远程仓库为了跟踪是谁提交的代码，需要提交这提供身份信息：邮箱和用户名。
  • 可以,本地Git的用户名和邮箱地址主要用于标识提交记录中的作者信息，而远程Git服务上的用户名通常用于身份验证和访问控制。此外
    • 如果Git配置的邮箱是GitHub中存在的邮箱，则commits里显示的是这个邮箱对应的账号
    • 如果Git配置的邮箱是一个在GitHub里不存在的邮箱，则commits里显示的是Git配置的用户名
